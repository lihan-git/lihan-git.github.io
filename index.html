Excution policy in windows powershell

windows powershell 的执行策略

powershell 相比于CMD命令行有很大的优势，特别是和linux命令结合方面，powershell 可以和Linux紧密的结合起来。所以现在是全面转向powershell的时候了！

powershell在Windows平台默认是不能执行脚本的，需要自行了解excution policy。下面从excution policy权限种类、权限范围和相关命令来介绍Windows平台下powershell的excution policy。

权限种类

AllSigned:
1.可以运行脚本

2.要求脚本和配置文件由可信任的publisher发布或者脚本由自己在local computer上写成

3.非信任来源的脚本运行给出提醒

Bypass:
运行所有脚本，没有提醒

RemoteSigned:
可以运行脚本，但需要网络下载数字签名，其中在local computer上写的脚本不需要数字签名

Restricted:
允许独立的命令，但不允许运行脚本

权限范围

MachinePolicy: 所有使用此计算机的人（组策略）

UserPolicy: 当前使用此计算机的人（组策略）

Process: 当前powershell会话

Current User: 当前使用此计算机的人

Local Machine: 所有使用此计算机的人[系统默认值]

权限更改命令

获取当前会话的执行权限信息

更改执行权限

移除执行权限

总结
一般来说，获取默认范围（local machine）的Bypass权限就可以了。
